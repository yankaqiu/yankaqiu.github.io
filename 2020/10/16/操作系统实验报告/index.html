二、设计说明

预备知识

查看sh.c代码以及一些重要函数的用法

首先，struct cmd 结构体用来记录命令的类型，下面的三类结构体execcmd，redircmd，pipecmd分别对应三条命令（可执行程序、重定向和管道命令）：

struct cmd {
  int type;          //  ' ' (exec), | (pipe), '<' or '>' for redirection
};

struct execcmd {   //exec cmd
  int type;              // ' '
  char *argv[MAXARGS];   // arguments to the command to be exec-ed
};

struct redircmd {   //redir cmd
  int type;          // < or > 
  struct cmd *cmd;   // the command to be run (e.g., an execcmd)
  char *file;        // the input/output file
  int flags;         // flags for open() indicating read or write
  int fd;            // the file descriptor number to use for the file
};

struct pipecmd {  //pipe cmd
  int type;          // |
  struct cmd *left;  // left side of pipe
  struct cmd *right; // right side of pipe
};

了解一下用到的一些重要函数的用法：

  1. srchr()函数

用命令man srchr查看:

The strchr() function returns a pointer to the first occurrence of the character c in the string s.

它的函数原型：

extern char *strchr(char* str,char character)

str为一个字符串的指针，character为一个待查找字符。功能：从字符串str中寻找字符character第一次出现的位置。

  1. man exec查看exec簇函数的用法，其中两个：
int execv(const char *path, char *const argv[]);

此函数会执行参数path 字符串所代表的文件路径，命令所需的参数以char *arg[]形式给出并且argv最后一个元素是NULL。

int execvp(const char *file, char *const argv[]);
  1. open函数：
int open(const char *pathname, int flags, mode_t mode)

参数：
1.pathname
  要打开或创建的目标文件
2.flags
  打开文件时，可以传入多个参数选项，用下面的
  一个或者多个常量进行“或”运算，构成falgs
  参数：
  O_RDONLY:   只读打开
  O_WRONLY:   只写打开
  O_RDWR:     读，写打开
这三个常量，必须制定一个且只能指定一个
  O_CREAT:    若文件不存在，则创建它，需要使
              用mode选项。来指明新文件的访问权限
  O_APPEND:   追加写，如果文件已经有内容，这次打开文件所
              写的数据附加到文件的末尾而不覆盖原来的内容

它的功能是打开和创建文件（建立一个文件描述符，其他的函数可以通过文件描述符对指定文件进行读取与写入的操作。）

  1. close函数

函数原型：

int close(int fd);

作用就是关闭一个已打开的文件

  1. dup函数

函数原型：

int dup(int oldfd);

其功能是create a copy of the file descriptor oldfd，即复制某个已经打开的文件描述符，得到一个新的描述符，这个新的描述符，也指向被复制描述符所指向的文件。

若成功，会返回当前系统可用的最小整数值。

其中需要补全的代码在函数runcmd中。

管道

管道是一种把两个进程之间的标准输入和标准输出连接起来的机制，从而提供一种让多个进程间通信的方法，当进程创建管道时，每次都需要提供两个文件描述符来操作管道。其中一个对管道进行写操作，另一个对管道进行读操作。对管道的读写与一般的IO系统函数一致，使用write()函数写入数据，使用read()读出数据。

int pipe(int filedes[2]);

//返回值：成功，返回0，否则返回-1。参数数组包含pipe使用的两个文件的描述符。fd[0]:读管道，fd[1]:写管道。

//必须在fork()中调用pipe()，否则子进程不会继承文件描述符。两个进程不共享祖先进程，就不能使用pipe。但是可以使用命名管道。

具体实现代码:

case '|':
    pcmd = (struct pipecmd*)cmd;
    fprintf(stderr, "pipe not implemented\n");
    // Your code here ...
    pipe(p);     //创建管道 
    if(fork1()==0){  //在子进程中
      close(1);
      dup(p[1]);    //将标准输出重定向至p[1]
      runcmd(pcmd->left);
    }
    else{
      close(p[1]);
      close(0);
      dup(p[0]);    //将标准输出重定向至p[0]
      wait(&r);
      runcmd(pcmd->right);
    }
    break;

第一次调用fork1()产生child process1，该进程用于运行pcmd->ledft指向的进程，然后将文件描述符1空余出来。 调用dup(p[1])把 child process 1的标准输出(文件描述符默认的是1)和管道的输出关联起来。

父进程需要接收子进程输出的数据，所以关闭父进程的p[1]，把进程的标准输入重定向至p[0]，然后在父进程中等待子进程输出完毕，然后执行管道右边的命令。

五、实验中遇到的问题及解决方法

  1. 文件权限问题

起初，在terminal中运行ls > y会报错

然后发现问题出在这里：

if(open(rcmd->file,rcmd->flags,0777)<0){
      fprintf(stderr,"fail to open the file %s\n",rcmd->file);
		  _exit(1);
    }

发现忘记指定文件的权限（是不认真读实验指导的导致的….）open函数的第三个参数mode指定新建文件的权限，mode是八进制权限码，其中0777表示文件所有者、该文件用户组、其他用户都有可读可写可执行的权限

  1. ls执行时的路径问题

在自己的shell上测试ls时，什么都不显示，然后发现是路径问题（这个实验题目上也提到了）

起初我的代码是：

if(execv(ecmd->argv[0],ecmd->argv)==-1){
     perror(ecmd->argv[0]);
}

解决的办法是改用execvp()函数：

if(execvp(ecmd->argv[0],ecmd->argv)==-1){
      perror(ecmd->argv[0]);
 }

execv()用来执行参数path 字符串所代表的文件路径,，execvp()会从PATH 环境变量所指的目录中查找符合参数file 的文件名, 找到后便执行该文件, 然后将第二个参数argv 传给该欲执行的文件。
